# インタラクティブなポートフォリオサイト

## コンセプト
- 「ドット絵」×「インタラクティブ（シェーダー）」をテーマにしたポートフォリオ。
- ゲーム開発の経験（GLSL/HLSL）を活かし、背景や作品展示にリッチなシェーダー表現を取り入れる。

## ページ構成・コンテンツ
1. **Home**
    - 名前、アイコン、ハンドルネーム
    - 技術スタック (Skills)
    - 経歴 (Career/About)
2. **2DWorks**
    - 普段制作しているドット絵作品のギャラリー
    - 画像のクリックで拡大表示などの機能
3. **ShaderWorks**
    - シェーダー作品の展示
    - 映像やGIFだけでなく、マウス操作等でインタラクティブに動くWebGLのライブデモも掲載

## インタラクション要件
- **背景シェーダー**: ドットの格子状のデザインをベースにし、マウスカーソルが合わさった部分が反応するエフェクト、またはGLSLによる流体/粒子(パーティクル)表現を背景に描画する。
- **UIとの連携**: React側で取得したマウス座標やスクロール量を `uniforms` としてGLSLシェーダーに渡し、リアルタイムに絵作りを変化させる。

## 技術スタック (確定)
- **フレームワーク**: React (構築ツール: Vite)
- **シェーダー・WebGL描画**: React Three Fiber (Three.js の React用ラッパー) + Drei
- **ルーティング**: React Router (複数ページの切り替え)
- **スタイリング**: Vanilla CSS または Tailwind CSS (ドット絵のレトロな雰囲気や、プレーンなUIを作成)



## FABRIKによるProcedualAnimation

SDFとFABRIKを用いて魚の様なアニメーションをマウスカーソルに追従させる。
### 使用するパラメータ
- 節の個数(末端含む。0が頭部)
- 各節の半径の大きさ
- 各節を結ぶ線分の長さ
- leapの方法(メソッド？)
- マウス追従の強さ
    - 頭部をマウスに追従させる方法として、基本的に頭部の移動は頭部とつながっている節に対して平行な移動と、頭部とつながっている節の角度回転移動の合成として考える(車みたいな)

## 背景シェーダー設計メモ（2026-06）

### 現在のページ順とインデックス
- 0: Home
- 1: About
- 2: 2D Works
- 3: Shader Works

### 設計方針（使い回し前提）
- 背景は「共通ベース（魚IKのSDF）」を中心に構成する。
- ページごとの差は、ベースに対する加工（ピクセル化、追加レイヤー、色調）として足す。
- トランジションは `u_pageIndex` の小数部 `t=fract(u_pageIndex)` を使い、`mix` で補間する。

### ページ別の役割
- Home:
    - 単体の球（頭）を表示する。
- About:
    - 現在の魚アニメーションを表示する。
- 2D Works:
    - Aboutベース（魚）をピクセル化した表現を使う。
    - 宇宙背景（星・星雲など）は後で追加する。
- Shader Works:
    - Aboutベース（魚）に加えて、粒子やセルオートマトン系の高密度表現を追加する。
    - GPGPUベースのシミュレーションは後で追加する。

### Home -> About 遷移仕様
- 遷移中は「頭（節0）以外の節半径」を `t` でスケールする。
- イメージ式:
    - `r0(t)=r0_base`
    - `ri(t)=ri_base * t` （i>0）
- これにより、`t=0` では球のみ、`t=1` で通常の魚形状に到達する。
